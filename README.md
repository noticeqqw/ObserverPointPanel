# Observer Pattern Panel

Реализация классического паттерна проектирования **Observer** на Java для управления точками на графической панели.

[![Java](https://img.shields.io/badge/Java-17-orange.svg)](https://www.oracle.com/java/)
[![Maven](https://img.shields.io/badge/Maven-3.8+-blue.svg)](https://maven.apache.org/)

## Описание

Этот проект демонстрирует паттерн Observer (Наблюдатель), который используется для установления связи один-ко-многим между объектами так, что при изменении состояния одного объекта все зависящие от него объекты уведомляются об этом и автоматически обновляются.

## Структура проекта

```
src/main/java/panel/points/
├── Main.java           - Точка входа приложения
├── Observable.java     - Издатель (слуховой объект)
├── Observer.java       - Интерфейс наблюдателя
└── PointsPanel.java    - Панель с точками и реализацией Observer
```

## Компоненты

### Observable.java
Класс, который хранит список наблюдателей и уведомляет их об изменениях состояния.

### Observer.java
Интерфейс, который должны реализовать все наблюдатели для получения уведомлений.

### PointsPanel.java
Реализация графической панели, которая:
- Действует как наблюдатель за точками
- Отображает точки на экране
- Реагирует на изменения в Observable

### Main.java
Запускает приложение и демонстрирует использование паттерна.

### Требования

- Java 8 или выше
- Maven 3.6+

## Как это работает

1. `Main` создает экземпляр `Observable`
2. `PointsPanel` регистрируется как наблюдатель
3. При добавлении новой точки в `Observable`:
   - Все зарегистрированные наблюдатели уведомляются
   - `PointsPanel` обновляет свой визуальный вывод
4. Паттерн позволяет слабую связанность между компонентами

## Паттерн Observer

**Преимущества:**
- Слабая связанность между издателем и подписчиками
- Динамическое добавление/удаление наблюдателей
- Поддержка broadcast-коммуникации

**Когда использовать:**
- Когда изменение в одном объекте требует изменения других, заранее неизвестное количество объектов
- Когда объект должен уведомлять другие объекты, не делая предположений об этих объектах

## Структура проекта

- `Main.java` - точка входа приложения
- `Observable.java` - слуховой объект
- `Observer.java` - интерфейс наблюдателя
